# Pokerface

Pokerface 是一个中文德州扑克小游戏，支持单机命令行、浏览器静态页面，以及 2.0 命令行 TCP 联机模式。项目不依赖数据库，联机房间默认只保存在服务端内存里。

## 在线游玩

GitHub Pages:

https://mihan-p.github.io/Pokerface/

## 命令行单机版

需要本机已安装 Node.js。

```bash
npm start
```

或：

```bash
node poker.js
```

启动后按提示设置人数、初始筹码、盲注、水下模式和 AI 难度。

## 命令行联机版

Pokerface 现在有两种联机方式：

- 正式客户端模式：服务端运行 `server`，房主/玩家运行本仓库里的 `create` / `join` 客户端。适合开发和完整协议测试。
- 纯终端模式：服务端运行 `telnet`，朋友只用系统终端里的 `nc` 或 `telnet` 连接，不需要拉代码。

如果朋友完全不想拉 Git 代码，使用“纯终端模式”。

## 纯终端模式，朋友不需要代码

服务器上启动：

```bash
node poker.js telnet --host 0.0.0.0 --port 8787
```

第一个连接进来的人会成为房主，并在连接里设置座位数、筹码、盲注、水下模式和 AI 难度。

朋友连接：

```bash
nc 服务器IP 8787
```

Windows 没有 `nc` 时可用：

```bash
telnet 服务器IP 8787
```

进入后输入昵称，再用 `sit 2` 入座。房主可以用 `bot add 3 AI-1 普通` 添加 AI，用 `start` 开始。

注意：`node poker.js create 服务器IP:8787` 不能连接纯终端模式。`create/join` 是 JSON 客户端，只能连接 `node poker.js server` 启动的 JSON 服务。

如果要让朋友公网访问，腾讯云安全组和服务器防火墙都要放行 TCP 8787。

## 正式客户端模式，需要玩家有 poker.js

### 启动服务端

在个人服务器上运行：

```bash
node poker.js server --host 0.0.0.0 --port 3000
```

也支持环境变量：

```bash
HOST=0.0.0.0 PORT=3000 node poker.js server
```

服务端启动后会输出管理口令。房主创建房间时需要输入这个口令。

### 创建房间

房主在自己的终端运行：

```bash
node poker.js create your-server.com:3000
```

按提示输入昵称、管理口令和房间配置。创建成功后终端会显示房间码和朋友加入命令。

### 加入房间

玩家运行：

```bash
node poker.js join your-server.com:3000 房间码
```

进入房间后可用 `入座 2` 或 `sit 2` 选择座位。

### 本机调试

一条命令启动本地服务端并创建房主客户端：

```bash
node poker.js host
```

该模式适合本机或内网测试，不作为正式跨网络朋友局方案。

## 联机房间命令

等待区玩家命令：

| 中文命令 | 快捷命令 | 含义 |
| --- | --- | --- |
| `入座 3` | `sit 3` | 坐到 3 号座位 |
| `离座` | `leave` | 离开当前座位 |
| `状态` | `s` | 刷新房间状态 |
| `退出` | `q` | 断开连接 |

房主命令：

| 中文命令 | 快捷命令 | 含义 |
| --- | --- | --- |
| `开始` | `start` | 开始第一手 |
| `下一手` | `next` | 单手结束后继续下一手 |
| `添加AI 3 AI-1 普通` | `bot add 3 AI-1 普通` | 在 3 号座位添加 AI |
| `移除AI 3` | `bot remove 3` | 移除 3 号座位 AI |
| `设置AI 3 小王 困难` | `bot config 3 小王 困难` | 修改 AI 名称和难度 |

牌局行动命令：

| 中文命令 | 快捷命令 | 含义 |
| --- | --- | --- |
| `弃牌` | `f` | 放弃本手牌 |
| `过牌` | `c` | 无需跟注时让过 |
| `跟注` | `c` | 补齐到当前最高下注 |
| `下注 50` | `b 50` | 下注 50 |
| `加注 120` | `r 120` | 加注到 120 |
| `全下` | `a` | 全下 |
| `状态` | `s` | 重新显示当前状态 |
| `退出` | `q` | 断开连接 |

## 联机安全边界

- 服务端是唯一权威，洗牌、发牌、AI 决策、行动校验和结算都在服务端执行。
- 客户端只提交命令，不直接修改牌局状态。
- 牌局中每个客户端只会收到自己的暗牌；其他玩家未公开暗牌不会出现在快照里。
- 断线后会保留座位，并通过本地 `.pokerface-sessions.json` 缓存的会话信息尝试重连。
- 当前行动玩家超时未行动时，服务端默认执行弃牌。
